## Directory Purposes

**`src/routes/`:**
- Purpose: One file per URL route; TanStack Router discovers and generates types automatically
- Contains: `__root.tsx` (shell), `index.tsx` (home), future routes added here
- Key files: `src/routes/__root.tsx`, `src/routes/index.tsx`
- Naming: `__root.tsx` for root layout, `index.tsx` for `/`, `about.tsx` for `/about`, `posts/$id.tsx` for `/posts/:id`

**`src/components/ui/`:**
- Purpose: Design system layer — reusable styled components installable via `bunx shadcn@latest add <component>`
- Contains: One file per component; each wraps `@base-ui/react` primitives with Tailwind + CVA
- Key files: `button.tsx`, `card.tsx`, `dropdown-menu.tsx`, `select.tsx`, `combobox.tsx`, `alert-dialog.tsx`, `field.tsx`, `input.tsx`, `badge.tsx`

**`src/components/` (root level):**
- Purpose: Feature and page-level component compositions built from UI primitives
- Contains: `component-example.tsx` (current app content), `example.tsx` (showcase layout helpers)

**`src/lib/`:**
- Purpose: Shared utility functions with no UI concerns
- Contains: `utils.ts` — the `cn()` class-merging helper
- Key files: `src/lib/utils.ts`

**`public/`:**
- Purpose: Static assets served directly (no processing)
- Contains: `favicon.ico`, `manifest.json`, logo PNGs, `robots.txt`
- Generated: No
- Committed: Yes

**`.planning/codebase/`:**
- Purpose: GSD codebase map documents consumed by `/gsd:plan-phase` and `/gsd:execute-phase`
- Generated: Yes (by `/gsd:map-codebase`)
- Committed: Yes

## Key File Locations

**Entry Points:**
- `src/routes/__root.tsx`: HTML shell, stylesheet injection, devtools — parent of all routes
- `src/routes/index.tsx`: Home route `/`
- `src/router.tsx`: Router factory used by SSR bootstrap and client hydration

**Configuration:**
- `vite.config.ts`: Build plugins (TanStack Start, Tailwind CSS, Nitro, path aliases)
- `tsconfig.json`: TypeScript strict mode, `@/*` → `./src/*` path alias
- `components.json`: shadcn CLI config (`base-nova` style, `src/styles.css` target, `@/components/ui` alias)
- `.prettierrc`: Formatting rules (single quotes, no semicolons, Tailwind class sort, import order)

**Core Logic:**
- `src/lib/utils.ts`: `cn()` — used in every component
- `src/styles.css`: All OKLCH design tokens, dark mode variants, font config, Tailwind theme

**Auto-generated (do not edit):**
- `src/routeTree.gen.ts`: Route manifest generated by `@tanstack/router-plugin` — regenerated on save/build

## Naming Conventions

**Files:**
- kebab-case for all source files: `component-example.tsx`, `alert-dialog.tsx`, `input-group.tsx`
- `__root.tsx` for TanStack Router root layout (double underscore prefix is TanStack convention)
- `index.tsx` for index routes

**Directories:**
- lowercase kebab-case: `src/components/ui/`, `src/lib/`

**Component exports:**
- PascalCase named exports: `export function Button(...)`, `export { Card, CardHeader }`
- No default exports on components

**CSS tokens:**
- Kebab-case CSS custom properties: `--primary`, `--muted-foreground`, `--sidebar-accent`
- Tailwind classes reference tokens via semantic names: `bg-primary`, `text-muted-foreground`

## Where to Add New Code

**New page/route:**
- Create `src/routes/<name>.tsx` with `createFileRoute('/<name>')({ component: PageComponent })`
- Route tree regenerates automatically on next dev server save

**New feature component (page-level composition):**
- Add to `src/components/<feature-name>.tsx`
- Import UI primitives from `src/components/ui/`
- Import `cn` from `@/lib/utils`

**New UI primitive (design system component):**
- Use `bunx shadcn@latest add <component>` — outputs to `src/components/ui/<component>.tsx`
- Do NOT manually create UI primitives; use the shadcn CLI to ensure correct `base-nova` style

**New utility function:**
- Add to `src/lib/utils.ts` if general-purpose
- Create `src/lib/<domain>.ts` for domain-specific helpers

**New styles / design tokens:**
- Add CSS custom properties to `src/styles.css` inside `:root` (light) and `.dark` blocks
- Expose to Tailwind via `@theme inline` block in the same file
- Never use raw OKLCH/hex values in component files — always reference semantic token classes

**Tests (when configured):**
- Co-locate with source: `src/components/<name>.test.tsx`
- No test framework is currently configured (vitest is installed but no test files exist)
